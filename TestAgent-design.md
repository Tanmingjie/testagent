# 产品设计
背景：AI Agent发展迅速，现已支持cdp操作浏览器（如playwright-cli），且有很多产品如testsprite等，已实现端到端的ai辅助测试执行。

目标：开发一个TestAgent（ai-agent）实现从文本用例->测试执行->测试报告，端到端的ai辅助测试。

# 第一阶段
核心能力：
1. 翻译文本用例。输入的文本用例必须满足规范：用例名称、预置条件、测试步骤、预期结果。由于每一个业务人员在书写文本用例的时候，业务人员的专业度是不一样的，文本用例的质量是高低不等的，翻译文本用例指的是将文本用例的预置条件、测试步骤、预期结果尽量专业化，可执行，agent可读。这一步需要考虑，不同的业务分别维护一份业务知识术语库。

2. 测试步骤、预期结果拆解。为了ai-agent可执行，将测试步骤、预期结果拆解，每一个step、需要单独对应一个预期结果，如此一来，可判断每一步的执行结果是否满足预期。


3. 测试执行阶段，ai-agent结合playwright-cli解释执行用例。注意：先通过cli解释执行用例，遇到问题通过agent自愈能力修复，最终执行完成，输出代码。

4. 测试结果报告

# 第二阶段
工程化，实现一个web界面：
1. 支持文本用例导入。
2. 文本用例树状结构显示
3. 可指定文本用例执行
4. 执行方式：一键执行，端到端；
5. 执行过程可视化，每一个测试步骤有截图或视频
